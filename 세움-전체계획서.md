## 1. 세움이란 무엇인가 — 정체성

세움은 **한국어로 작성하는 시스템 프로그래밍 언어**다. 네 가지 정체성이 모두 핵심이며, 어느 하나만 빠져도 세움이 아니다.

| 축 | 의미 |
|---|---|
| **① 한국어 일급 시민** | Unicode 한글 식별자, 한국어 키워드·에러 메시지, `(ㄱ:10, ㅅ:20)` 자모 좌표 |
| **② 시스템 언어** | C와 Rust 사이 포지셔닝. RAM/CPU/VGA 직접 접근, 베어메탈 가능. `위험 { }` 블록으로 저수준 노출 |
| **③ OS-like 런타임 (나의세상)** | 전통 OS와 다른 형태. Smalltalk 이미지 + WebAssembly 하이브리드 |
| **④ '담음' 컨테이너** | `.docx`/`.mp4`를 대체하는 자기-설명 파일 표준. 코드+상태+미디어가 한 컨테이너에 |

**가까운 선례**: Smalltalk-80 (언어=환경=이미지) + WebAssembly/WASI + Inferno/Limbo (언어가 OS) + 한국어 일급. 어느 하나의 fork가 아니라 **합성**.

**왜 만드는가**: "한국어로 컴퓨터의 모든 것을 다시 설계하는" 문화/플랫폼 프로젝트. 단순 교육 도구가 아니다.

## 3. 완성된 것 (v0.0 ~ v0.3) — 예시와 함께

### 3.1 v0.0 — MVP 슬라이스 ✅

첫 목표: `보여주기(지금).` 이 세움 표준시(SST)를 출력한다.

```seum
// examples/hello.세움 — 세움의 첫 프로그램
가져오기(시간).
보여주기(지금).
```

- **트리워킹 인터프리터**: lexer → parser → interpreter
- **SST (세움 표준시)**: epoch = 1945-08-15 06:00 KST (독도의 첫 일출). Unix 초로 -769402800
- **한글 식별자 1급**, 문장 종결자 `.` (마침표)
- 명세 v1.0 의 4 acceptance 통과

### 3.2 v0.1 — 정리 슬라이스 ✅

기능 추가 없이 *기초를 다진* 슬라이스.

- Catch2 테스트 프레임워크를 `third_party/`에 amalgamated로 vendoring (인터넷·git 의존 0)
- `보여주기` 의 출력을 의존성 주입(`OutputSink`)으로 — `freopen` 폐기
- `value.h` 인터페이스 좁히기 — `is_*`/`as_*` 헬퍼만 외부 노출
- Windows 빌드 래퍼 `build.bat`/`test.bat`/`run.bat` + vswhere 자동 MSVC 환경

```seum
// examples/annyeong.세움 — 한글 식별자 + 변수 선언
변수 안녕 = "세움".
보여주기(안녕).
```

### 3.3 v0.2 — 언어 확장 + 바이트코드 VM ✅

명세 자체 리뷰에서 v0.0 언어에 Bool·연산자·제어구문이 없음이 드러나, v0.2를 "VM 골격"에서 **언어 확장 + VM 메이저**로 재정의.

| 단계 | 내용 |
|---|---|
| v0.2a | IR + Bytecode + VM 골격, 양 경로 회귀 |
| v0.2b | Bool 타입 + 비교·논리 연산 + `만약`/`아니면` |
| v0.2c | 산술·비교 연산자 + 문자열 `+` 연결 |
| v0.2d | 반복문 `반복 N번`, `동안` |
| v0.2e | 함수 정의 + 재귀 + `--vm` 기본 전환 |

**실행 모델**: 스택 머신, 3-주소 IR (SSA 아님), JIT 없음. opcode는 한글 1급 표면 + 영문 enum 내부.

```seum
// 분기 — acceptance 5
만약 (3 > 2) 이면 {
    보여주기("크다").
} 아니면 {
    보여주기("작다").
}

// 반복 — acceptance 9
반복 3번 {
    보여주기("야!").
}

// 재귀 — acceptance 12, 5! = 120
함수 팩토리얼(n) -> 결과 {
    만약 (n <= 1) 이면 { 돌려주기 1. }
    돌려주기 n * 팩토리얼(n - 1).
}
보여주기(팩토리얼(5)).
```

**양 경로 의무 (결정 #32)**: 트리워킹(`--tree`)과 VM(`--vm`)이 *같은 언어를 같은 결과로* 실행. `--tree`는 회귀 비교용으로 영구 유지.

### 3.4 v0.3 — '담음' 컨테이너 + 그릇 ≡ 담음 ✅

가장 큰 정체성 통합 슬라이스.

**'담음'의 정의 (결정 #45)** — "세움세상의 평화를 위한 그릇". 평화 = (i) 기존 시스템과의 호환 + (ii) 한글 콘텐츠의 정확한 표현·압축.

**그릇 ≡ 담음 (결정 #46)** — *세움 세계에는 두 종류 객체가 없고 하나만 있다 — 담음*. 사용자 프로그램, 표준 그릇, 나의세상 인스턴스 모두 동일한 `.담음` 구조.

| 단계 | 내용 |
|---|---|
| v0.3a | binary 포맷 reader/writer |
| v0.3b | bytecode ↔ binary, `세움 빌드` 명령어 |
| v0.3c | 자모 Huffman 압축 + 한글텍스트 chunk |
| v0.3d | `.txt ↔ .담음` 양방향 변환 |
| v0.3e | `위험` 식 + `지연값` Getter + native bridge + `가져오기(.담음)` |

**v0.3e 의 도약** — builtin 그릇을 `.세움` 소스로 재작성하며 자기-호스팅의 첫 걸음을 뗐다:

```seum
// lib/보여주기.세움 — C++ builtin의 .세움 동등물 (결정 #46, #58)
함수 보여주기(값) -> 없음 {
    위험 { native_보여주기(값) }.
}
```

```seum
// lib/시간.세움 (발췌) — 'native'는 시스템 초 하나뿐, SST 포맷은 전부 .세움 안
지연값 지금() -> 결과 {
    돌려주기 형식화_SST(위험 { native_시각_시스템_초() }).
}
```

`지연값`은 0-arg Getter 1급 키워드. `위험 { }`은 호스트 native 함수를 부르는 *식* 형태 (블록 형태는 v0.5).

## 5. '담음' 컨테이너 포맷

`.세움` (UTF-8 텍스트 소스) → 컴파일 → `.담음` (binary 산출물). Java의 `.java → .jar` 대응.

### 5.1 구조

```
[ 헤더 16 bytes ]  magic="SEUM" + version + flags + chunk_count + CRC32
[ Chunk Table ]    chunk당 16 bytes (type·compression·offset·size)
[ Chunk Bodies ]   각 16-byte 정렬 (베어메탈 부트로더 친화)
```

### 5.2 chunk 타입 — 한글 자모가 식별자

| 자모 | chunk | 내용 | 분류 |
|---|---|---|---|
| ㄱ | 코드 | bytecode::Program 직렬화 | 필수 |
| ㄴ | 이름 | 시그니처·export 목록 | 필수 |
| ㄷ | 메타 | 작성자·시각(SST)·버전·체크섬 | 필수 |
| ㅁ | 상태 | 상수 풀 + 변수 binding | 필수 |
| ㅎ | 한글텍스트 | 자모 Huffman 압축 + 형태소 토큰 | 필수 |
| ㅂ | 미디어 | raw bytes | 옵셔널 |
| ㅅ | 소스 | `.세움` 원본 | 옵셔널 |
| ㅇ | 링크 | 외부 `.담음` 경로 | 옵셔널 |

예약: ㄹ·ㅈ·ㅊ·ㅋ·ㅌ·ㅍ (v0.4+ 의미 부여).

### 5.3 명령어

```
세움 빌드 hello.세움                  → hello.담음
세움 실행 hello.담음
세움 변환 hello.txt -> hello.담음      (양방향, round-trip 비트 동일)
```

`가져오기(시간)`은 `<lib_dir>/시간.담음`을 검색·로드. lib_dir 우선순위: 환경변수 `SEUM_LIB_PATH` → 실행파일 옆 `lib/` → `~/.seum/lib/`.

#### R2 종합 점검에서 짚은 사항

- **함수 1급(62)의 근거 정정**: R1은 "`누르면 -> 함수()` 콜백 위해 필수"라 했으나, 원본 예시는 모두 인라인 식(`누르면 -> 알림(...)`). 정확한 근거는 "콜백 블록을 1급 함수값으로 *내부 저장·전달*하는 구현 수단 + `변수 f = 함수…` 활성화". 결정은 유지, 근거만 교체.
- **현실성 경고 (⚠️)**: v0.4는 역대 최대 슬라이스 — 네이티브 GUI + 실제 이벤트 루프 + 실시간 게임 루프 + HTTP 클라/서버 + RPC + 암호화 + 그릇 3 + 코어 확장 3 + 누적 ~26 acceptance. v0.3(견적 10~17주)의 2~3배 규모. 압축 패턴은 명세를 촘촘히 잠갔기에 유효하나, **sub-slice를 작게 자르는 것이 유일한 tractability 열쇠**.
- **공유 인프라**: GUI native bridge는 게임(v0.4b)·웹(v0.4c)이 공유 → v0.4b에서 한 번 구축, v0.4c에서 재사용.

## 8. 작업 방식 — software-fundamentals 7단계

모든 non-trivial 코딩 작업의 디폴트 워크플로:

```
1. Grill Me      의도 추궁 (Why/Who/What/Where Not/When Stop)
2. Spec Lock     명세 합의 + Ubiquitous Language
3. Module Design Deep Module 설계 + Vertical Slice
4. Dependency Check  의존성 3가지 질문
5. Build         TDD (RED → GREEN → REFACTOR)
6. Review        의식적 코드 리뷰
7. Daily Design  매일 시스템 설계에 투자
```

**1~4단계가 5단계보다 길어야 정상이다.** v0.2·v0.3의 한 자리수 압축은 이 원칙의 증명.

## 3. v0.2 모듈 골격 (Deep Module)

세 개의 새 모듈. 각각 인터페이스 좁고 구현 깊음을 목표. 외부 의존성 추가 없음 (잠긴 결정 #13).

### 3.1 `src/ir.{h,cc}` — 3-주소 중간 표현

**공개 인터페이스 (전부)**:
```cpp
namespace seum::ir {
    struct Program;                  // IR 프로그램 (함수 + 상수 풀)
    Program lower(const seum::Program& ast);  // AST → IR
    std::string disassemble(const Program& ir);  // 사람용 한글 출력
}  // namespace seum::ir
```

**왜 Deep**: 외부는 4 심볼만 본다. 3-주소 명령어 enum/operand 타입/SSA 흉내 검토 등은 전부 `ir.cc` 내부.

**의존성**: `ast.h` (입력) 만. AST 의 새 `is_*`/`as_*` 헬퍼만 사용 (잠긴 결정 #22 의 사전 조건).

### 3.2 `src/bytecode.{h,cc}` — 바이트코드 표현 + 디스어셈블

**공개 인터페이스 (전부)**:
```cpp
namespace seum::bytecode {
    struct Chunk;                    // 인스트럭션 + 상수 풀 + 라인 정보
    Chunk compile(const ir::Program& ir);
    std::string disassemble(const Chunk& chunk);  // 한글 opcode 표시
    std::u32string opcode_name(uint8_t op);       // 한글 표면 이름
}  // namespace seum::bytecode
```

**왜 Deep**: opcode enum, 인코딩 (가변/고정), 상수 풀 자료구조, 라인 매핑 — 전부 내부. 표면은 4 심볼.

**bytecode_names.h** (인라인 표): 영문 enum ↔ 한글 표면의 단일 소스. 디스어셈블·에러·문서 모두 이 표 통과.

### 3.3 `src/vm.{h,cc}` — 스택 머신 실행기

**공개 인터페이스 (전부)**:
```cpp
namespace seum::vm {
    int run(const bytecode::Chunk& chunk,
            seum::Environment& env,
            seum::modules::OutputSink sink = seum::modules::default_stdout_sink());
}  // namespace seum::vm
```

**왜 Deep**: 스택, 프레임, 호출 규약, 디스패치 루프 — 전부 `vm.cc` 내부. 표면은 1 함수.

**의존성**: `bytecode.h`, `environment.h`, `modules/builtin.h` (`OutputSink`). 새 외부 의존 0.

### 3.4 모듈 호출 그래프

```
main.cc
  ├── --tree → interpreter::evaluate(ast, env)               [v0.0 유지]
  └── --vm   → vm::run(bytecode::compile(ir::lower(ast)), env)  [v0.2 신규]
```

`interpreter.{h,cc}` 와 `vm.{h,cc}` 는 형제. 한쪽이 다른 쪽을 알지 못함. 공유는 `Environment` + `OutputSink` 두 지점만.

## 4. 인스트럭션 셋 초안 (28개, v0.0 커버용 최소)

**원칙**: v0.0 의 4개 acceptance를 모두 커버할 수 있는 최소 집합. v0.4 (그릇·계약)에서 30~50개로 확장.

| 분류 | 한글 표면 | 영문 enum | 의미 |
|---|---|---|---|
| 상수 | `정수상수` | `BC_CONST_INT` | 상수 풀 idx 의 정수를 스택 푸시 |
| 상수 | `문자열상수` | `BC_CONST_STR` | 상수 풀 idx 의 문자열을 푸시 |
| 상수 | `참` | `BC_TRUE` | true 푸시 |
| 상수 | `거짓` | `BC_FALSE` | false 푸시 |
| 상수 | `없음` | `BC_NIL` | nil 푸시 |
| 스택 | `복제` | `BC_DUP` | 스택 top 복제 |
| 스택 | `버리기` | `BC_DROP` | 스택 top 제거 |
| 스택 | `바꾸기` | `BC_SWAP` | 위 두 원소 교환 |
| 산술 | `더하기` | `BC_ADD` | a, b → a+b |
| 산술 | `뺄셈` | `BC_SUB` | a, b → a-b |
| 산술 | `곱하기` | `BC_MUL` | a, b → a*b |
| 산술 | `나누기` | `BC_DIV` | a, b → a/b |
| 산술 | `나머지` | `BC_MOD` | a, b → a%b |
| 산술 | `음수` | `BC_NEG` | a → -a |
| 비교 | `같음` | `BC_EQ` | a, b → bool |
| 비교 | `다름` | `BC_NE` | a, b → bool |
| 비교 | `작음` | `BC_LT` | a, b → bool |
| 비교 | `큼` | `BC_GT` | a, b → bool |
| 논리 | `아닌` | `BC_NOT` | a → !a |
| 변수 | `지역가져오기` | `BC_LOAD_LOCAL` | 지역 idx → 스택 |
| 변수 | `지역두기` | `BC_STORE_LOCAL` | 스택 → 지역 idx |
| 변수 | `전역가져오기` | `BC_LOAD_GLOBAL` | 이름 idx → 스택 (환경 조회) |
| 변수 | `전역두기` | `BC_STORE_GLOBAL` | 스택 → 환경 (이름 idx) |
| 분기 | `뛰기` | `BC_JMP` | 절대/상대 offset |
| 분기 | `거짓이면뛰기` | `BC_JFZ` | top 이 거짓이면 분기 |
| 호출 | `부르기` | `BC_CALL` | argc 와 함께 호출 |
| 호출 | `돌려주기` | `BC_RET` | 프레임 반환 |
| 그릇 | `가져오기` | `BC_IMPORT` | 그릇 이름 idx → 환경 등록 |
| 그 외 | `종료` | `BC_HALT` | 인터프리터 정지 |

**총 29개.** v0.0 acceptance 4개:
1. `가져오기(시간). 보여주기(지금).` → `IMPORT 시간 · LOAD_GLOBAL 보여주기 · LOAD_GLOBAL 지금 · CALL 1 · CALL 1 · HALT`
2. `변수 안녕 = "세움". 보여주기(안녕).` → `CONST_STR 0 · STORE_GLOBAL 안녕 · LOAD_GLOBAL 보여주기 · LOAD_GLOBAL 안녕 · CALL 1 · HALT`
3. `가져오기(시간)` 없는 `지금` 사용 → `LOAD_GLOBAL 지금` 에서 환경 조회 실패 → 기존 `SeumError` 흐름
4. 위 셋 자동 테스트

**참이면뛰기/큼/거짓** 등은 v0.0 예제엔 안 쓰이지만 v0.2 ship 시 포함. v0.4 그릇·계약에서 `MCALL`/`NEW_CONTRACT` 등 확장 예정.

## 5. Ubiquitous Language 확장 (v0.2)

| 한국어 | 정의 | 영문 | 어디서 |
|---|---|---|---|
| **중간표현 (IR)** | 3-주소 코드 형태의 중간 단계 | intermediate representation | `src/ir.h` |
| **묶음 (Chunk)** | 바이트코드 + 상수 풀 + 라인 정보 한 덩어리 | bytecode chunk | `bytecode::Chunk` |
| **상수 풀** | 묶음 내부의 정수·문자열·이름 리터럴 표 | constant pool | `Chunk` 내부 |
| **풀어쓰기 (disassemble)** | 묶음을 사람이 읽을 한글 표기로 변환 | disassembly | `disassemble()` |
| **명령 (opcode)** | 한 바이트 단위 인스트럭션 | opcode | enum + 한글 표면 |
| **프레임** | 한 함수 호출의 스택 영역 | call frame | `vm.cc` 내부 |
| **VM 경로 / 트리 경로** | `--vm` / `--tree` 두 실행 모드 | VM / tree-walking path | CLI 플래그 |

## 6. v0.2 슬라이스 세분화

각 슬라이스는 끝에서 끝까지 동작 가능한 vertical slice. RED → GREEN → REFACTOR.

### v0.2a — IR + 디스어셈블 (1~2주)
- `ir::lower(ast)` 구현
- `ir::disassemble()` 한글 출력
- 통합 테스트: v0.0 4 acceptance 의 AST → IR 변환이 안정적인 한글 dump
- **ship 기준**: 디스어셈블 출력이 PR 리뷰에서 사람이 읽고 검증 가능

### v0.2b — 바이트코드 컴파일러 + 디스어셈블 (1~2주)
- `bytecode::compile(ir)` 구현
- 한글 opcode 표면 디스어셈블
- 통합 테스트: 4 acceptance 의 bytecode dump 골든 비교
- **ship 기준**: 골든 4개 모두 안정. 인스트럭션 셋이 29개를 넘지 않음(또는 명세 v1.3 으로 갱신)

### v0.2c — VM 실행기 + CLI 통합 (2~3주)
- `vm::run()` 디스패치 루프
- `main.cc` 의 `--vm`/`--tree` 플래그
- 통합 테스트: v0.0 4 acceptance 가 양쪽 경로에서 동일 출력
- 마이크로 벤치: `보여주기(지금).` 1만 회 실행이 VM 경로 ≥ 2배 빠름 (없으면 v1.3 으로 미루기)
- **ship 기준**: 4 acceptance 양쪽 통과, 트리 경로 회귀 0

**v0.2 종합 ship**: 명세 v1.3 발행 (없으면 v1.2 유지). 로드맵 v0.2 ✅ 표시. v0.2c 후 다음 슬라이스는 v0.3 '담음' 진입 + Grill Me 2라운드 필수.

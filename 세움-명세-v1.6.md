## 6. v0.4+ 진입 의무 (그릇·계약 슬라이스)

명세 v1.0 §6 의 v0.4 = 도메인 그릇들 (`웹` / `게임` / `금융`). 진입 의무:

1. **Grill Me 2라운드 필수** (v0.3 진입 시처럼). 각 그릇이 *세움 정체성과 어떻게 정합* 하는지 잠금:
   - `웹` 그릇: HTML/CSS/JS 트랜스파일? 자체 렌더링? — v0.3 결정 #56 의 *HTTP 의존* 와 충돌 가능
   - `게임` 그릇: 한글 자모 좌표 `(ㄱ:10, ㅅ:20)` 1급 시민화 — *언어 syntax 확장* 필요
   - `금융` 그릇: 자산 1급, `받아서 ->` 파이프 시멘틱, 계약 syntax — v0.0 명세 §3 의 `계약` 키워드 활성
2. **v0.4 진입 전 발견된 백로그**:
   - 함수 1급 시민화 (`변수 f = 함수…`) — v0.4 grunt
   - 중첩 함수 / 클로저
   - `-> 결과` 의 *실제 시멘틱 부여* (v0.2 명세 v1.4 의 약속) — 계약·그릇에서 결과 명명 의미
   - 자모 chunk types 추가 매핑 (예약 6개 ㄹ/ㅈ/ㅊ/ㅋ/ㅌ/ㅍ)
   - `--순수` flag (builtin 우선 vs .담음 우선)
   - `lib/시간.담음` 의 자동 회귀 (cmake `SEUM_SOURCE_DIR` define)
   - 짧은 텍스트 자모 Huffman 의 *팽창* 문제 — static frequency table 또는 small-text fast-path
   - `--vm` / `--tree` 양립 — `위험` 식과 native bridge 가 *양 경로 의무* (#32) 위배 가능성 검토 필요
3. **v0.5 의 `위험` *블록* 형태 완성** — v0.3e 의 *식* 형태 위에 덧붙임 (결정 #60)

## 8. v0.3 종료 — 의식적 짧은 회고

> 이 슬라이스는 *원본 비전 (프로그래밍예시.md)* 에 없던 *'담음'* 컨테이너를 **세움세상의 평화를 위한 그릇** 으로 *발견*·*구현* 했다. 사용자가 "평화" 라는 단어로 정의한 두 함의 — (i) 기존 시스템과의 호환 + (ii) **한글 콘텐츠의 정확한 표현** — 이 단순한 파일 포맷 이상의 *철학적 기반* 을 제공했다.
>
> "그릇 ≡ 담음" (결정 #46) 의 통합은 Smalltalk image + Lisp homoiconicity + WebAssembly module 의 *한국어 재구성*. v0.3e 의 acceptance 16/17 가 이 정체성을 *작동* 으로 증명했다.
>
> `위험` 식 부분 도입 (결정 #58) + `지연값` Getter 1급 (결정 #59) 은 v0.5 시스템 슬라이스의 *부분 선결*. 자기-호스팅 v1.0 의 길이 *원본 명세 너머* 로 한 걸음 나아갔다.

## 1. 핵심 모델 — 통화 발행 = 언어 담보 검증

**모든 통화 발행은 언어가 담보를 검증한다.** 담보가 두 종류:

| 발행 종류 | 담보 (언어가 검증) | 한도 |
|---|---|---|
| **첫 발행** | 발행자가 **담보로 건 특정 BTC** (금고 전체 잔액 아님) | 그 BTC × 페그. 예: 1 BTC → 정확히 10만 마실 |
| **대출 발행 = 신용 창조** | 계약의 **지급준비율** | 통화승수(= 1/지급준비율)가 신용 창조 총량 상한 |

`understanding_bank_loans_highschool.md` 그대로:
- 원리 ① 대출 = 화폐 창조 → **대출 발행**.
- 원리 ② 지급준비율·통화승수 → 대출 발행 한도가 곧 `준비금 ÷ 지급준비율`.

## 2. 언어 primitive (C++ 변경)

### 2.1 통화 선언 + 페그
```
통화 마실 = BTC / 100000.
```
새 통화 `마실`. 페그: **1 BTC = 100000 마실**. 마실은 정수 단위 (소수 없음).

### 2.2 발행 연산 `발행 <금액> 담보 <대상>`
1급 식. 검증 통과 시 *새로 창조된* 통화값 반환, 실패 시 한국어 에러.
- 담보 = **BTC 자산** → 언어: `발행 금액 ≤ 금액(담보 BTC) × 100000`. (첫 발행 — 그 BTC 만큼만.)
- 담보 = **계약** → 언어: 지급준비율 한도 검사 (§3). (대출 발행.)

### 2.3 자산 빼기 `-=`
a-2 `+=` 대칭. 상환·출금. 결과 음수 → "음수 잔액 금지" 에러 (결정 64).

### 2.4 `금액(자산) -> 정수`
금융 그릇 내장. 자산의 최소단위 정수 추출. 페그·한도 계산용.

## 3. 대출 발행 한도 = 지급준비율 〈검토 요망〉

대출 발행의 담보 = "계약 내용" = 계약이 선언한 **지급준비율**.

계약은 `지급준비율: N` (정수 %, 예: 10 = 10%) 을 선언한다. `발행 X 담보 (이 계약)` 시
언어가 다음을 검사한다:

```
금액(발행량) + 금액(X)  ≤  금액(금고) × 100000 × 100 ÷ 지급준비율
```

- `금액(금고) × 100000` = 준비금(BTC)을 마실로 환산.
- `× 100 ÷ 지급준비율` = 통화승수 (지급준비율 10% → 10배).
- 예: 금고 1 BTC, 지급준비율 10 → 최대 1,000,000 마실까지 신용 창조.

→ **"계약 내용 검증 = 지급준비율"** 이 핵심. 이 형태가 맞는지 확인 부탁드립니다.

## 4. 마실은행.세움 — 예시 프로그램 (초안)

```seum
가져오기(금융).
통화 마실 = BTC / 100000.

계약 세움은행 {
    자산 금고: BTC
    자산 발행량: 마실
    지급준비율: 10                  // 10% — 통화승수 10배

    // 고객이 BTC 를 맡김 — 금고에 누적
    함수 예금(받은돈: BTC) 받아서 -> 금고 {}

    // 첫 발행 — 맡긴 그 BTC 만큼만 마실 발행 (1:1 페그, BTC 담보).
    함수 첫발행(맡긴: BTC) -> 결과 {
        세움은행.금고 += 맡긴.
        돌려주기 발행 마실(금액(맡긴) * 100000) 담보 맡긴.
    }

    // 대출 — 신용 창조. 계약(지급준비율) 담보로 마실을 무에서 창조.
    함수 대출(원금: 마실) -> 결과 {
        변수 빌린돈 = 발행 원금 담보 세움은행.
        세움은행.발행량 += 빌린돈.
        돌려주기 빌린돈.
    }
}

변수 첫돈 = 세움은행.첫발행(BTC(1)).      // 1 BTC → 10만 마실
보여주기(첫돈).                            // 마실(100000)
변수 대출금 = 세움은행.대출(마실(500000)). // 신용 창조 (10배 한도 안)
보여주기(대출금).                          // 마실(500000)
보여주기(세움은행.발행량).
```

## 5. 잠길 결정 후보

- D-A: 통화 발행은 언어 담보 검증 필수 — 무담보 발행 불가.
- D-B: `통화 이름 = 기준 / 비율` — 새 통화 선언 + 페그.
- D-C: `발행 <금액> 담보 <대상>` 1급 식. 담보=BTC → `≤ 담보 × 페그`. 담보=계약 → `발행량+X ≤ 준비금÷지급준비율`.
- D-D: 계약이 `지급준비율: N` 선언 → 대출 발행 한도의 근거.
- D-E: `-=` 자산 빼기 + `금액(자산)` 내장. 소수 미도입 — 정수 스케일.

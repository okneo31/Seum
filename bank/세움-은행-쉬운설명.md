# 세움 은행 — 그림으로 보는 쉬운 설명

> 초등학생도 한눈에. **표와 그림** 으로 풀어 봅니다.

세움은 한국어로 컴퓨터를 움직이는 프로그래밍 언어입니다.
그 세움으로 진짜 은행처럼 돈을 만들고, 빌려주고, 갚을 수 있습니다.

---

## 1. 마실 — 세움 나라의 돈

나라마다 쓰는 돈이 다릅니다.

| 나라 | 돈 이름 |
|------|---------|
| 한국 | 원 |
| 미국 | 달러 |
| **세움** | **마실** |

세움 나라의 돈은 "마실"입니다.

---

## 2. 은행 = 돈을 담는 상자

은행은 돈을 담아 두는 큰 상자입니다.
세움에서는 이 상자를 **계약** 이라고 부릅니다.

```
   ┌──────── 은행 ────────┐
   │                      │
   │   [ 돈 주머니 ]       │
   │     0 마실           │
   │                      │
   └──────────────────────┘
       처음엔 텅 비어 있음
```

---

## 3. 대출의 비밀 — 돈은 "빌려줄 때 새로 생긴다"

여기가 가장 신기한 부분입니다.

| | 보통 생각 | 진짜로 일어나는 일 |
|--|-----------|--------------------|
| 대출하면? | 은행 돈이 **줄어든다** | 새 돈이 **생긴다** |
| 어떻게? | 금고에서 돈을 꺼냄 | 장부에 숫자를 적음 |

은행은 100만 마실을 빌려줄 때 금고에서 돈을 꺼내지 않습니다.
그냥 **"100만 마실"이라고 적습니다.** 그 순간 새 돈이 태어납니다.

```
   대출 전     은행 [        0 마실 ]
                       │
                       │  "100만 마실" 적기 (새로 만듦!)
                       ▼
   대출 후     은행 [  1,000,000 마실 ]
```

> 게임에서 운영자가 명령어로 아이템을 만들어 주는 것과 똑같습니다.
> 이것을 **신용 창조** 라고 합니다.

그래서 대출 후의 돈은 — 빼기가 아니라 **더하기**:

```
    원래 돈        빌린 돈        대출 후 돈
   ┌───────┐     ┌───────┐     ┌───────┐
   │   0   │  +  │ 100만 │  =  │ 100만 │
   └───────┘     └───────┘     └───────┘
       T      +      L      =      R
```

---

## 4. 이자 — 빌린 값을 얹어서 갚기

돈을 빌리면 — 빌린 돈(원금)만 갚는 게 아닙니다.
빌려 쓴 값으로 **이자** 를 조금 더 얹어 갚습니다.

이자는 **이자율** 로 정해집니다. 세움은 이자율을 **BP** 라는 정수로 적습니다.

| 이자율 | 세움의 BP |
|:---:|:---:|
| 1% | 100 BP |
| 4.5% | 450 BP |
| 10% | 1000 BP |

100만 마실을 연 4.5%(450 BP)로 빌리면:

| | 마실 |
|---|---:|
| 빌린 돈 (원금) | 1,000,000 |
| 이자 (4.5%) | + 45,000 |
| **갚을 돈 합계** | **1,045,000** |

계산은 곱하기·나누기뿐 — 어렵지 않습니다:

```
   이자  =  원금 × 이자율(BP) ÷ 10000
        =  1,000,000 × 450 ÷ 10000
        =  45,000 마실
```

---

## 5. 상환 — 원금과 이자를 나눠, 기간 동안 갚기

빌린 돈은 한 번에 다 갚지 않습니다. **기간** 을 정해 나눠 갚고,
갚을 때마다 **원금**(빌린 돈)과 **이자**(빌린 값)를 **함께** 냅니다.

예 — 120만 마실을 **4개월** 동안, 연 4.5%로 빌렸다면:

- 매달 갚는 **원금** = 120만 ÷ 4 = **30만** 씩 (똑같이 나눔)
- 매달 갚는 **이자** = **남은 빚** 에 붙음 → 빚이 줄면 이자도 줄어듦
  (연 4.5% 는 12달로 나눠 → 한 달 0.375%)

| 기간 | 갚는 원금 | 갚는 이자 | 합계 | 남은 빚 |
|:---:|---:|---:|---:|---:|
| 1개월 | 300,000 | 4,500 | 304,500 | 900,000 |
| 2개월 | 300,000 | 3,375 | 303,375 | 600,000 |
| 3개월 | 300,000 | 2,250 | 302,250 | 300,000 |
| 4개월 | 300,000 | 1,125 | 301,125 | 0 |
| **합계** | **1,200,000** | **11,250** | **1,211,250** | — |

- **원금** 은 매달 똑같이 30만씩 — 4번에 나눠 120만 전부 갚음.
- **이자** 는 남은 빚이 줄수록 함께 줄어듭니다 (4,500 → 1,125).
- 4개월 뒤 — 남은 빚이 **0**. 다 갚았습니다.

이렇게 원금을 똑같이 나눠 갚는 방법을 **원금균등상환** 이라고 합니다.

---

## 6. 연체이자 — 늦으면 더 무겁게

갚기로 한 날에 못 갚으면 — **연체** 입니다.
연체하면 평소보다 **더 높은 이자율** 이 붙습니다. 이것이 **연체이자율** 입니다.

| 상황 | 이자율 | 100만에 붙는 이자 |
|---|:---:|:---:|
| 제때 갚음 (약정) | 4.5% (450 BP) | 45,000 마실 |
| 늦게 갚음 (연체) | **7.5%** (750 BP) | **75,000 마실** |

연체이자율 = 약정이자율 + **연체가산**(벌점) → 4.5% + 3% = 7.5%.

> 늦을수록 빚이 더 빨리 불어납니다. 그래서 제때 갚는 게 중요합니다.

## 7. 복리의 마법 — 이자가 이자를 낳는다

이자에는 두 종류가 있습니다.

| | 단리 | 복리 |
|---|---|---|
| 이자를 어디에 붙이나? | **원금에만** | **원금 + 그동안 쌓인 이자** |
| 그래서? | 이자가 늘 똑같음 | 이자가 점점 커짐 |

복리는 — 받은 이자가 다음 해엔 **자기 자신도 이자를 낳습니다.**

```
   1년차:  원금 100  →  +이자 10  →  110
   2년차:  110       →  +이자 11  →  121    ← 이자가 10 아닌 11!
   3년차:  121       →  +이자 12  →  133    ← 점점 커진다
```

100만 마실을 연 10%로 묻어 두면 — 시간이 갈수록 단리와 복리의 차이가 폭발합니다:

| 햇수 | 단리 | 복리 | 복리가 몇 배? |
|:---:|---:|---:|:---:|
| 1년 | 110만 | 110만 | — |
| 10년 | 200만 | 259만 | 1.3배 |
| 20년 | 300만 | 673만 | 2.2배 |
| 30년 | 400만 | **1,745만** | **4.4배** |

> 30년 뒤 — 단리는 원금의 **4배**, 복리는 **17배**.
> 시간이 길수록 복리는 무섭게 커집니다. 이것이 **복리의 마법** 입니다.

---

## 8. 복식부기 — 원금도 이자도, 양쪽에 똑같이

진짜 은행은 무슨 일이 생기든 **장부 두 칸에 똑같은 금액**을 적습니다.
원금만이 아니라 — **이자도** 똑같이 두 칸에 적습니다.

```
     대출 1건  =  원금 기입 + 이자 기입,  둘 다 양쪽에

   ┌────────────────────────┬────────────────────────┐
   │   왼쪽: 은행이 받을 것   │   오른쪽: 은행이 만든 것 │
   ├────────────────────────┼────────────────────────┤
   │  대출채권   + 100만      │  고객 예금   + 100만     │  ← 원금
   │  미수이자   +   4.5만    │  이자수익    +   4.5만   │  ← 이자
   ├────────────────────────┼────────────────────────┤
   │  합계       + 104.5만    │  합계        + 104.5만   │
   └────────────────────────┴────────────────────────┘
              왼쪽 합계 = 오른쪽 합계  →  항상 균형
```

- **원금** — 대출채권(받을 원금) ↔ 고객 예금(줄 원금), 각 100만.
- **이자** — 미수이자(받을 이자) ↔ 이자수익(은행이 번 돈), 각 4.5만.
- 원금이든 이자든 **반드시 양쪽에 똑같이** → 장부는 1마실도 안 틀립니다.

## 9. 세움 코드는? — 딱 이만큼

이 모든 게 세움에선 짧은 한국어 코드입니다. 한 줄씩이면 끝납니다.

| 하는 일 | 세움 코드 (한 줄) |
|---|---|
| 돈 만들기 | `통화 마실.` |
| 대출 (빌려주기) | `함수 대출(원금) 받아서 -> 전체계좌합 {}` |
| 상환 (갚기) | `함수 상환(갚은돈) -> 없음 { 전체계좌합 -= 갚은돈. }` |
| 이자 계산 | `함수 이자(원금) -> 결과 { 돌려주기 마실(금액(원금)*이자율/10000). }` |
| 연체이자 | `함수 연체이자(금액) -> 결과 { 돌려주기 마실(금액(금액)*연체이자율/10000). }` |
| 복리 거듭제곱 | `반복 연수 번 { 변수 잔액 = 잔액 * (10000+이자율) / 10000. }` |

`받아서 -> 전체계좌합` 은 "받은 원금을 전체계좌합에 더한다"는 뜻 — 그게 곧 `R = T + L` 입니다.

---

## 한 줄 요약

```
   대출  →  돈을 옮김이 아니라 새로 만듦    ( R = T + L )
   이자  →  빌린 값을 원금에 얹어 갚음
   복리  →  이자가 이자를 낳아 시간 따라 폭발
```

> 세움 은행은 — 돈(마실)을 만들고 · 대출로 새 돈을 창조하고(R=T+L) ·
> **이자**를 얹어 갚게 하며 · 늦으면 연체이자 · 길게 두면 복리로 불립니다.
> 그리고 이 모든 게 한국어 코드 몇 줄.

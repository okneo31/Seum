## v0.2 종료 — `세움-명세-v1.4.md` 발행. 12 acceptance 양 경로 통과. v0.3 진입 전 Grill Me 2라운드 의무.

잠금된 핵심 (결정 16–33):
- 스택 머신, 3-주소 IR, JIT 없음, image 는 v0.3 '담음'과 통합
- 한글 opcode 1급 (`복사`/`부르기`/`만약 … 이면`…), 영문 enum 내부 (`BC_DUP`/`BC_CALL`)
- 트리워킹·VM 양 경로 동일 언어 실행 (양립). 기본은 v0.2a~d 동안 `--tree`, v0.2e ship 시 `--vm` 전환
- Bool 1급 (`참`/`거짓`), 연산자는 기호만 (한국어 정체성은 식별자·키워드·에러로 유지)
- 견적 정직: 한 슬라이스 미끄러지면 두 배. v0.2a 완료 시 견적 갱신.

## v0.4 — 도메인 그릇들 (예상 각 3~5주)

### `웹` 그릇
```
화면구성 { 제목, 글, 버튼, 누르면 -> ... }
```
- HTML/CSS/JS 트랜스파일? 자체 렌더링?
- 기존 `프로그래밍예시.md`에 있는 `버튼/누르면` 시멘틱 구현

### `게임` 그릇
```
게임세상 { 배경, 캐릭터, 누르면 -> 이동 }
캐릭터 용사 { 위치: (ㄱ: 10, ㅅ: 20) }
```
- 한글 자모 좌표 (`ㄱ`=가로, `ㅅ`=세로) 1급 시민화
- 게임 루프 추상화
- 렌더링 백엔드 결정

### `금융` 그릇
```
계약 나의은행 { 자산 금고: BTC; 함수 예금(받은돈: BTC) 받아서 -> 금고 {} }
```
- 자산 타입 1급
- `받아서 ->` 파이프 시멘틱
- 실제 블록체인 연결? 시뮬레이션?

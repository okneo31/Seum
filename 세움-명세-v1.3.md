## 2. v0.2 재정의 — 신규 결정 (25–33)

v1.2 자체 검토에서 F1·F2 가 드러남: v0.0 의 Value 시스템에 Bool 없음, lexer/parser 에 연산자·제어구문 없음. 따라서 *바이트코드 VM*만으로는 v0.2 셋이 비어있게 됨. 사용자 의사 결정으로 **v0.2 = 언어 확장 + VM** 메이저 슬라이스로 재정의.

| # | 항목 | 결정 |
|---|---|---|
| 25 | **Bool 타입** | `BoolValue` 를 `Value` variant 에 추가. `is_bool` / `as_bool` 헬퍼. `render_value` 한국어로 `참` / `거짓` 출력. |
| 26 | **연산자 표기** | 모두 **기호** — 산술 `+ - * / %`, 비교 `== != < > <= >=`, 논리 `! && \|\|`. 한국어 키워드 변형 없음. 한국어 정체성은 식별자·키워드·에러 메시지·디스어셈블에서 유지. |
| 27 | **진리값 리터럴** | **`참` / `거짓`** 키워드. 식별자로 사용 불가 (예약). |
| 28 | **if/else** | **`만약 (조건) 이면 { 본문 } 아니면 { 본문 }`**. 괄호+조사 형식. `아니면` 절은 선택. else-if 는 `아니면 만약 (...) 이면 { ... }` 중첩. |
| 29 | **반복문** | 두 가지 동시 도입: `반복 N번 { 본문 }` (정수 식 N회) 및 `동안 (조건) { 본문 }`. `번` 은 키워드, `반복`/`동안` 도 키워드. |
| 30 | **함수 정의** | `함수 이름(매개1, 매개2) -> 결과 { 본문 }`. 명시적 `돌려주기 식.` 으로 반환. `-> 결과` 는 시그니처 문서 마커이며 v0.2 에서는 *결과명 미사용* (v0.4 계약·그릇 슬라이스에서 의미 부여). 반환값 없는 함수는 `-> 없음`. |
| 31 | **문자열 연결** | `+` 오버로딩. 양변 모두 `string` 일 때만. 정수+문자열·문자열+정수 모두 **타입 에러** (자동 변환 없음). |
| 32 | **양 경로 의무** | 트리워킹·VM 둘 다 위 모든 신규 문법을 지원. CLI `--vm` / `--tree` 출력 동일. (결정 #21 확장.) |
| 33 | **v0.2 단계화 재설계** | v0.2a~e 5 슬라이스. §6 참조. 견적 12~20주 (사용자 페이스에서 6~10주 가능, §8 참조). |

**갱신 규칙**: 25~33 변경 시 명세 v1.4. v0.2 본 빌드 중 신규 발견 시 errata 또는 v1.4 백로그 누적.

## 3. 토큰 / AST 확장 (v0.2 신규)

### 3.1 새 토큰 종류
```cpp
enum class TokenKind {
    // 기존 …
    Plus, Minus, Star, Slash, Percent,
    EqEq, BangEq, Lt, Gt, LtEq, GtEq,
    Bang, AmpAmp, PipePipe,
    Arrow,                   // ->
    // 기존 Keyword 안에서 식별: 만약, 이면, 아니면, 반복, 번, 동안, 함수, 돌려주기, 참, 거짓, 없음
};
```

`Arrow` 는 `-`/`>` 두 글자 매치. 한국어 키워드는 lexer 가 식별자 자리에서 lookup 후 `Keyword` 토큰화.

### 3.2 새 AST 노드
```cpp
struct BoolLitExpr   { bool value; Position pos; };
struct BinaryExpr    { std::u32string op; Expr lhs; Expr rhs; Position pos; };
struct UnaryExpr     { std::u32string op; Expr operand; Position pos; };
struct IfStmt        { Expr cond; std::vector<Stmt> then_body; std::vector<Stmt> else_body; Position pos; };
struct WhileStmt     { Expr cond; std::vector<Stmt> body; Position pos; };
struct RepeatStmt    { Expr count; std::vector<Stmt> body; Position pos; };
struct FuncDeclStmt  { std::u32string name; std::vector<std::u32string> params;
                       std::u32string result_name; std::vector<Stmt> body; Position pos; };
struct ReturnStmt    { Expr value; Position pos; };
```

`Expr` variant 에 `BoolLitExpr`, `BinaryExpr`, `UnaryExpr` 추가.
`Stmt` variant 에 `IfStmt`, `WhileStmt`, `RepeatStmt`, `FuncDeclStmt`, `ReturnStmt` 추가.

`is_*` / `as_*` 헬퍼는 v0.1 잔여 정리에서 도입한 패턴 그대로 확장.

## 4. 인스트럭션 셋 v1.3 (39개)

**v0.2 전체 문법 커버.** F2 정직성 회복.

| 분류 | 한글 표면 | 영문 enum | 의미 |
|---|---|---|---|
| 상수 | `정수상수` | `BC_CONST_INT` | 상수 풀 idx 의 정수 푸시 |
| 상수 | `문자열상수` | `BC_CONST_STR` | 상수 풀 idx 의 문자열 푸시 |
| 상수 | `참` | `BC_TRUE` | `참` 푸시 (Bool) |
| 상수 | `거짓` | `BC_FALSE` | `거짓` 푸시 (Bool) |
| 상수 | `없음` | `BC_NIL` | monostate 푸시 |
| 스택 | `복사` | `BC_DUP` | top 복사 (F10 의 `복제`→`복사` 결정) |
| 스택 | `버리기` | `BC_DROP` | top 제거 |
| 스택 | `바꾸기` | `BC_SWAP` | 위 두 원소 교환 |
| 산술 | `더하기` | `BC_ADD` | a, b → a+b (문자열일 때 결합, 정수일 때 합) |
| 산술 | `빼기` | `BC_SUB` | a, b → a-b |
| 산술 | `곱하기` | `BC_MUL` | a, b → a*b |
| 산술 | `나누기` | `BC_DIV` | a, b → a/b. 정수 나눗셈. 0 나눗셈 = 런타임 에러 |
| 산술 | `나머지` | `BC_MOD` | a, b → a%b |
| 산술 | `음수` | `BC_NEG` | a → -a |
| 비교 | `같음` | `BC_EQ` | a, b → Bool |
| 비교 | `다름` | `BC_NE` | a, b → Bool |
| 비교 | `작음` | `BC_LT` | a, b → Bool |
| 비교 | `큼` | `BC_GT` | a, b → Bool |
| 비교 | `작거나같음` | `BC_LE` | a, b → Bool |
| 비교 | `크거나같음` | `BC_GE` | a, b → Bool |
| 논리 | `아닌` | `BC_NOT` | a → !a (Bool) |
| 논리 | `그리고` | `BC_AND` | 단락 평가는 분기로 처리. 이 opcode 는 비단락 변형이며 실제 컴파일러는 `BC_JFZ` 로 풀어씀. (※ v0.2c 에서 도입 vs 보류 결정.) |
| 논리 | `또는` | `BC_OR` | 동상. |
| 변수 | `지역가져오기` | `BC_LOAD_LOCAL` | 지역 idx → 스택 |
| 변수 | `지역두기` | `BC_STORE_LOCAL` | 스택 → 지역 idx |
| 변수 | `전역가져오기` | `BC_LOAD_GLOBAL` | 이름 idx → 스택 (환경 조회, **GetterValue 자동 발화** — F4) |
| 변수 | `전역두기` | `BC_STORE_GLOBAL` | 스택 → 환경 |
| 분기 | `뛰기` | `BC_JMP` | 상대 offset |
| 분기 | `거짓이면뛰기` | `BC_JFZ` | top Bool 이 거짓이면 분기. top 소비. |
| 분기 | `참이면뛰기` | `BC_JNZ` | top Bool 이 참이면 분기. top 소비. |
| 호출 | `부르기` | `BC_CALL` | argc 명시. 스택에 [callee, arg1, …, argN] → 결과 1 푸시 |
| 호출 | `돌려주기` | `BC_RET` | 현 프레임 종료. top 을 호출자 스택에 푸시 |
| 그릇 | `가져오기` | `BC_IMPORT` | 그릇 이름 idx → 환경 등록 |
| 그 외 | `종료` | `BC_HALT` | 인터프리터 정지. **모든 Chunk 끝에 의무** (F7) |

**총 34개.** §1.2 의 29 보다 5 증가: 비교 4 신규(`LE`/`GE`/`AND`/`OR`)·`JNZ`. F1 의 Bool 합법화로 비교·논리·분기 활성.

## 5. Ubiquitous Language 추가 (v0.2)

기존 (v1.0/v1.1/v1.2) 어휘 그대로 + 아래.

| 한국어 | 정의 | 영문 |
|---|---|---|
| **참 / 거짓** | Bool 리터럴, 조건의 결과 | `true` / `false` |
| **만약 … 이면 … 아니면 …** | 조건 분기 구문 | if / else |
| **반복 … 번** | 정수 N 회 반복 | repeat N times |
| **동안 (…) {…}** | 조건 참인 동안 반복 | while |
| **함수 … (…) -> … {…}** | 함수 정의 | function declaration |
| **돌려주기 …** | 반환 | return |
| **없음** | nil/monostate 의 1급 이름 | nil |
| **명령 위치** | 프로그램 카운터 (디버그·에러 메시지) | program counter |
| **프레임** | 호출 한 회의 스택 영역 | call frame |
| **재귀** | 함수가 자신을 호출 | recursion |

## 6. v0.2 슬라이스 재설계 (a~e)

각 슬라이스는 vertical: 토큰→AST→트리워킹→VM→테스트 끝에서 끝까지. RED→GREEN→REFACTOR.

| 슬라이스 | 범위 | 양 경로 가산물 | 예상 |
|---|---|---|---|
| **v0.2a** | **IR + Bytecode + VM 골격** (현재 v0.0 언어만) | 트리·VM 둘 다 v0.0 4 acceptance 통과. CLI `--tree` 기본, `--vm` opt-in. | 2~3주 |
| **v0.2b** | **Bool + 비교 + 논리 + `만약`/`아니면`** | 트리·VM 둘 다 if/else 실행. acceptance 5 추가: `만약 (1 == 1) 이면 { 보여주기("같음"). } 아니면 { 보여주기("다름"). }` | 2~4주 |
| **v0.2c** | **산술·비교 연산자 + 문자열 연결** | acceptance 6: `보여주기(1 + 2 * 3).` → `7`. acceptance 7: `보여주기("세움" + " 안녕").` → `세움 안녕`. acceptance 8: 0 나눗셈 한국어 에러. | 2~3주 |
| **v0.2d** | **반복문 (`반복 N번`, `동안`)** | acceptance 9: `반복 3번 { 보여주기("야!"). }`. acceptance 10: 카운터 `동안` 루프. | 1~2주 |
| **v0.2e** | **함수 정의 + 재귀 + `--vm` 기본 전환** | acceptance 11: `함수 더하기(a, b) -> 합 { 돌려주기 a + b. }`. acceptance 12: 재귀 팩토리얼. v0.2e ship 시 CLI 기본 `--vm` (F5). 명세 v1.4 발행. | 3~5주 |

**합계 견적**: 10~17주 풀타임. 사용자 실 페이스(v0.0+v0.1 = 1일)는 슬라이스가 압축적이라 가능한 빠른 페이스는 **6~10주**. 정직 단서: 한 슬라이스 미끄러지면 두 배 가능. v0.2a 완료 시점에 v1.4 견적 갱신.

**ship 기준 패턴**: 각 슬라이스 = (1) 양 경로 동일 출력 회귀 (2) 새 acceptance 골든 비교 (3) 명세 △ 또는 v1.x 발행.

## 9. v0.0 acceptance 의 정정 인스트럭션 sequence (F3)

**Acceptance 1**: `가져오기(시간). 보여주기(지금).`

```
가져오기   "시간"
전역가져오기 "보여주기"
전역가져오기 "지금"       ; GetterValue 자동 발화 → TimeValue 푸시
부르기 1                   ; 보여주기(TimeValue)
버리기                     ; ExprStmt 결과 폐기
종료
```

**Acceptance 2**: `변수 안녕 = "세움". 보여주기(안녕).`

```
문자열상수 0              ; "세움"
전역두기 "안녕"
전역가져오기 "보여주기"
전역가져오기 "안녕"
부르기 1
버리기
종료
```

**Acceptance 3**: `가져오기(시간)` 없이 `지금`. → `전역가져오기 "지금"` 디스패치 중 런타임 에러. 컴파일 시점에는 통과 (F15).

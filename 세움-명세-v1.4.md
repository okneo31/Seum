## 3. v0.2e 토큰·AST·opcode 추가

### 3.1 토큰
- `Arrow`: `->` (2 글자)
- 키워드: `함수`, `돌려주기`

### 3.2 AST
- `FuncDeclStmt { name, params, result_name, body, pos }` (Stmt variant, unique_ptr)
- `ReturnStmt   { value, has_value, pos }` (Stmt variant, unique_ptr)
- `result_name` 은 v0.2e 에서 **시그니처 문서 마커** (결정 #30). v0.4 계약·그릇 슬라이스에서 의미 부여.

### 3.3 IR Op (4 추가)
- `PUSH_NIL` — `돌려주기.` 의 implicit 없음
- `LOAD_LOCAL` / `STORE_LOCAL` — 지역 (operand = local_idx)
- `RET` — 프레임 종료, top 을 호출자에게

### 3.4 Bytecode Opcode (5 추가)
- `PUSH_NIL = 0x05`
- `LOAD_LOCAL = 0x12` (4B local_idx)
- `STORE_LOCAL = 0x13` (4B local_idx)
- `RET = 0x21` (operand 없음)

**v0.2 누적 opcode**: 34개 (HALT 포함). 명세 §4 v1.3 표 (29개) + v0.2e의 5개 신규.

## 4. 함수 호출 시멘틱

### 4.1 양 경로 공통 의무 (결정 #32 확장)
- `함수 f(a, b) -> r { ... }` 정의 후 `f(x, y)` 호출 시 *동일 결과*
- 인자 수 불일치 = 한국어 SeumError "인자 개수 불일치: 기대 N / 받음 M"
- 함수 호출은 새 지역 스코프 생성. 매개변수는 첫 N 개 지역으로 배치.
- 함수 본문에서 외부 글로벌 함수 호출 가능 (전역 fallback)
- 재귀 자기 호출 가능

### 4.2 트리 경로 구현 (요약)
- 트리 인터프리터에서 FuncDeclStmt 평가 = CallableValue 생성하여 env 에 등록
- CallableValue 의 fn 클로저 = 새 Environment(parent=정의 시 env), 매개변수 바인딩, body 평가
- `돌려주기` = `ReturnSignal` 예외로 흐름 단축

### 4.3 VM 경로 구현 (요약)
- `ir::lower` 가 top-level FuncDeclStmt 마다 `ir::Function` 발행. Main 은 별도 Function.
- `bytecode::compile` 이 각 Function 의 Chunk 컴파일. 함수 Chunk 끝에 implicit `PUSH_NIL + RET`.
- `vm::run` 이 시작 시 모든 사용자 함수를 env 에 CallableValue 로 등록 → main Chunk 실행
- CALL 디스패치 시 callable->fn 호출 → 함수 Chunk 실행 (locals 배열 = 매개변수 + 본문 var)
- RET = top 을 반환, 프레임 종료

## 5. Ubiquitous Language 추가 (v0.2e)

|---|---|---|
| **돌려주기** | 함수 반환 | return |

| **프레임** | 함수 호출 한 회의 지역 + 스택 영역 | call frame |
